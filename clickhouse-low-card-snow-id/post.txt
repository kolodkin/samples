As we scaled out our cyber enterprise solution on ClickHouse,
we found some quick performance wins using LowCardinality for repetitive string columns and UInt64 with Snowflake IDs instead of String. UUID also great alternative l, just avoid Strings for IDs. 

Code examples:

```
CREATE TABLE events (
    status LowCardinality(String),
    created_at DateTime
) ENGINE = MergeTree() ORDER BY created_at;
```

```
CREATE TABLE events (
    id UInt64 DEFAULT generateSnowflakeID(),
    user_id UInt64,
    ...
) ENGINE = MergeTree() ORDER BY id;
```
